5 Kafka Concepts to Know (Before Starting)
CONCEPT
WHAT IT MEANS
Broker
The Kafka server — stores and serves messages
Topic
A named channel — like a database table for messages
Partition
A topic is split into partitions for parallel processing
Producer
Your code that sends messages to a topic
Consumer
Your code that reads messages from a topic
⚠️ Ignore For NowReplication factor, ISR, exactly-once semantics, consumer group rebalancing — all important eventually, but learn them ONLY when your project forces you to. Not before.
